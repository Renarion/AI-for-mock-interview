# Быстрая настройка LLM

## 1. API-ключ (только здесь, не в Git)

1. Откройте файл **`backend/.env`** (создайте из `backend/.env.example`, если его нет).
2. Добавьте строку:
   ```env
   OPENAI_API_KEY=sk-...ваш_ключ...
   ```
3. **Не коммитьте** `.env` и не вставляйте ключ в `llm_config.yaml`, если репозиторий публичный.
4. Если ключ когда-либо попал в чат или в код — **отзовите его** в [OpenAI API keys](https://platform.openai.com/api-keys) и создайте новый.

Опционально: непустой `LLM_PROVIDER=openai` в том же `.env` переопределяет провайдера из YAML.

## 2. Модель и температура

Файл **`backend/app/llm_config.yaml`**:

- **`models.openai`** — идентификатор модели для официального OpenAI API (например `gpt-4.1-nano`).  
  Если указали вид `openai/gpt-4.1-nano`, префикс `openai/` при запросе отбрасывается автоматически.
- **`temperature.full_interview`** — температура для финального разбора.
- **`max_tokens.openai_full_interview`** — лимит ответа (при длинном фидбеке при необходимости увеличьте).

После любых правок YAML **перезапустите бэкенд** (uvicorn / контейнер).

## 3. Системный промпт

В том же **`llm_config.yaml`** правьте блок **`prompts.full_interview_system`**.

Важно: фронтенд и API ожидают от модели **один JSON-объект** с полями `task_feedbacks`, `overall_score`, и т.д. (см. текст в системном промпте). Если убрать JSON и оставить только свободный текст, страница итогового отчёта перестанет корректно заполняться.

Содержательный разбор по задачам переносите в поле **`detailed_feedback`** каждого элемента `task_feedbacks` (там поддерживается Markdown на фронте).

## 4. Как «вшиваются» вопросы и ответы

Они **не хранятся статично в YAML**: при завершении интервью бэкенд собирает **user-сообщение** в `app/services/llm.py` (функция сборки тела запроса) и подставляет реальные тексты вопросов, ответов кандидата и эталонов из сессии. Менять шаблон нумерации блоков — правка **`llm.py`**, не YAML.

## 5. Проверка

1. Перезапуск бэкенда.
2. Пройти интервью до конца на сайте.
3. При ошибке смотреть логи бэкенда и текст фидбека «Ошибка LLM: …» на странице отчёта.

Подробнее о ключах: корневой **`KEYS_SETUP_GUIDE.md`**, общий обзор: **`README.md`**.
